# LF-Todo v1.4.0 Stable 最终封版计划

## 封版定位

`v1.4.0 Stable` 是 LF-Todo 进入低维护阶段前的最终稳定版。这个版本不再追求功能扩张，而是把现有个人使用链路打磨成稳定、可信、可安装、可备份、可恢复、可复查的本地桌面待办工具。

核心判断：

> LF-Todo 对个人使用已经足够完整，最后一轮迭代应聚焦封版质量，而不是继续扩展产品边界。

## 产品边界

### 保留并稳定的能力

- 待办增删改查、完成状态、优先级、截止时间。
- 分类、系统视图、暂存区、全部、已归档。
- 子任务、任务备注、拖拽排序。
- 分类内搜索、全局搜索与结果跳转。
- Quick Add 悬浮录入。
- 番茄钟与任务绑定专注。
- 本地快捷键、全局快捷键、自定义绑定。
- 数据导出、覆盖恢复、合并导入。
- 托盘、置顶、关闭到托盘、窗口记忆、开机自启。
- 自动归档、归档浏览与恢复。
- 应用内自动更新。

### 明确不纳入封版的能力

- 云同步、账号系统、多设备协同。
- 移动端 App。
- AI 任务整理。
- 看板、甘特图、复杂日历视图。
- 插件系统。
- 多主题、深色模式。
- 复杂统计报表。
- 任务模板、周期任务、标签系统。

这些方向可以保留在未来想法中，但不应进入 `v1.4.0 Stable`。

## 最后一轮迭代目标

### P0：稳定性封版

1. 跑通 `pnpm verify:agent:full`。
2. 确认 `pnpm build:win` 可正常产出 Windows 安装包。
3. 确认安装、启动、关闭到托盘、退出、置顶、最小化、自动更新状态不阻断主流程。
4. 检查关键危险操作确认文案：删除分类、删除任务、归档、恢复备份、合并导入。
5. 检查可见中文文案是否存在乱码。

### P0：数据安全与恢复

1. 备份导出成功后必须有明确提示。
2. 覆盖恢复前必须明确提醒会清空当前待办、分类和归档数据。
3. 合并导入前必须明确提醒可能出现重复内容。
4. README 必须写清楚数据存储在系统用户数据目录下，主数据库文件为 `lite-todo.db`。
5. README 必须写清楚迁移到新电脑的推荐方式是导出 JSON 备份后导入。

### P1：UI/UX 收口

1. 保持浅色 Arctic Blue、无边框窗口、紧凑高密度方向。
2. 所有 hover 不使用上浮、抬升或位移。
3. 弹层、右键菜单、Toast、下拉统一使用设计层级变量。
4. 任务卡片操作可发现，但不干扰正文阅读。
5. “全部”视图的拖拽柄以禁用光标提醒不支持排序。
6. 空状态文案继续指向下一步动作。

### P1：文档与发布

1. 更新 `package.json` 版本为 `1.4.0`。
2. 更新 README 的版本定位、功能说明、数据安全说明、开发与验证命令。
3. 新增 `docs/RELEASE_NOTES_v1.4.0.md`。
4. 更新 `docs/PROJECT_DOCS_SUMMARY.md`，标记当前版本进入 Stable 封版。
5. 保留 `docs/UI_UX_REFACTOR_PLAN.md` 作为 UI 决策记录。

## 封版验收清单

### 自动验证

```bash
pnpm verify:agent:standard
pnpm verify:agent:full
```

### 构建验证

```bash
pnpm build:win
```

### 手动验证

- 创建、编辑、完成、删除任务。
- 新建、重命名、删除分类。
- 创建、完成、排序子任务。
- 分类内搜索、全局搜索、结果跳转。
- Quick Add 录入到默认分类与指定分类。
- 番茄钟自由专注与任务绑定专注。
- 批量归档、单条归档、归档恢复。
- 导出备份、覆盖恢复、合并导入。
- 设置页快捷键修改与恢复默认。
- 托盘、置顶、关闭到托盘、退出。
- 安装包启动与版本显示。

## 发布建议

版本名称：`v1.4.0 Stable`

建议发布说明标题：

> LF-Todo v1.4.0 Stable：个人本地待办工具封版稳定版

建议发布原则：

- `v1.4.0 Stable` 后进入低维护模式。
- 后续只处理数据安全、启动失败、安装打包失败、严重交互阻断和依赖安全升级。
- 新功能想法统一进入待办池，不默认进入开发。

## 低维护规则

封版后只接受以下类型修改：

1. 数据丢失、数据无法恢复、备份失败。
2. 应用无法启动、无法安装、无法打包。
3. 严重 Electron 系统集成问题。
4. 自动更新严重失败。
5. 明显 UI 阻断或文案乱码。
6. 安全依赖升级。

其他需求默认暂缓。
