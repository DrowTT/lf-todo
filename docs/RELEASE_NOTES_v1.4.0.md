# LF-Todo v1.4.0 Stable 发布说明

## 版本定位

`v1.4.0 Stable` 是 LF-Todo 的个人使用封版稳定版。该版本聚焦本地优先、轻量、高效、可备份、可恢复，不再扩张为云端协作或多平台产品。

## 主要能力

- 待办、子任务、备注、优先级、截止时间。
- 分类、全部、暂存区、已归档等系统视图。
- 分类内搜索与全局搜索。
- Quick Add 悬浮快速录入，支持 `#分类名`。
- 番茄钟与任务绑定专注。
- 任务与子任务拖拽排序。
- 本地快捷键、全局快捷键与自定义绑定。
- 数据导出、覆盖恢复、合并导入。
- 托盘常驻、窗口置顶、关闭到托盘、开机自启、窗口记忆。
- 已完成任务自动归档与归档恢复。
- 应用内自动更新。

## 本轮封版改进

- 统一 Arctic Blue 设计令牌、弹层层级、阴影和 Tailwind 主题基准。
- 收敛任务头部、任务移动菜单和任务卡片交互细节。
- 优化任务拖拽柄：常规视图保持清晰可见，“全部”视图显示禁用光标提示不支持排序。
- 缩短列表内搜索展开与聚焦延迟。
- 修复主任务头部可见乱码。
- 补齐 UI/UX 重构方案与最终封版计划文档。

## 数据安全说明

- LF-Todo 是本地优先应用，不需要注册登录，不上传待办数据。
- 主数据库文件位于系统用户数据目录，文件名为 `lite-todo.db`。
- 推荐通过设置页的“导出备份”生成 JSON 文件，再在新设备中使用“恢复”或“合并导入”。
- “恢复备份”会覆盖当前待办、分类和归档数据；“合并导入”会保留当前数据，但可能产生重复任务。

## 验证建议

发布前建议执行：

```bash
pnpm verify:agent:full
pnpm build:win
```

并手动验证安装、启动、托盘、置顶、Quick Add、搜索、备份恢复、归档恢复和番茄钟主流程。

## 后续维护策略

`v1.4.0 Stable` 后进入低维护模式。后续只优先处理数据安全、启动失败、安装打包失败、严重交互阻断、自动更新阻断和依赖安全升级。
