# LF-Todo UI/UX 优化与重构方案

## 背景

LF-Todo 当前已经从「极简待办」扩展为包含分类、子任务、归档、全局搜索、Quick Add、番茄钟、快捷键、备份与自动更新的本地桌面效率工具。现有 Arctic Blue 方向正确，但界面复杂度已经超过早期原型，主要问题变成：视觉令牌分散、信息层级不够稳定、部分高频操作可发现性不足、弹层层级缺少统一规范。

本方案目标不是更换产品风格，而是在保留「极简、浅色 Arctic Blue、无边框窗口、紧凑高密度」的前提下，建立可持续演进的 UI/UX 基准。

## 设计原则

1. **极简但不隐藏关键操作**：高频操作应在 hover 前有轻量提示，危险操作保持低干扰但可发现。
2. **高密度但保证可读性**：继续保持桌面效率工具的紧凑布局，但正文、标题、辅助信息要有稳定层级。
3. **浅色 Arctic Blue 一致性**：主色维持清爽蓝灰体系，少量橙色只用于番茄钟、时间或强调反馈。
4. **原位 hover**：所有 hover 效果禁止上浮、抬升或位移，只使用颜色、边框、阴影、透明度变化。
5. **低风险渐进改造**：优先改设计令牌、样式层和可发现性，不改变数据流和业务逻辑。
6. **键盘与动效友好**：保留清晰 focus ring，尊重 `prefers-reduced-motion`，减少非必要连续动画。

## 设计系统基准

### 色彩

- 背景：窗口底色使用冷蓝灰，主内容区使用近白色，侧栏使用浅蓝灰。
- 主色：继续使用蓝色作为任务管理、搜索、焦点与选中状态主色。
- 功能色：成功为绿色，危险为红色，提醒/番茄钟为橙色。
- 玻璃态：仅用于弹层、上下文菜单、轻浮层，不滥用于所有卡片。

### 字号

- 正文继续保持紧凑，但需要明确 `xs / sm / md / lg / xl / 2xl` 的语义。
- 任务正文与页面标题必须优先保证可读性，不应被徽章和辅助信息抢占层级。
- 数字统计使用 tabular nums，提升计数和倒计时稳定性。

### 圆角与阴影

- 小控件：8px 左右。
- 卡片/输入区：12px 至 16px。
- 弹层：16px 以上，配合轻玻璃和较柔阴影。
- 阴影只表达层级，不表达 hover 位移。

### 层级

建立统一层级语义：

- `base`：普通内容。
- `sticky`：吸附头部、拖拽提示。
- `dropdown`：下拉选择器。
- `context-menu`：右键菜单。
- `modal`：确认框、全局搜索。
- `toast`：全局提示。

## 优先级计划

### P0：必须优先处理

1. **可见文案乱码修复**
   - 修复标题栏、活动栏、任务头部、任务卡片中的可见乱码。
   - 这类问题直接影响可信度，优先级最高。

2. **主界面层级收敛**
   - 任务头部需要清晰区分：当前视图标题、局部搜索、待办计数、批量归档。
   - 任务卡片操作按钮需要提高可发现性，避免所有操作完全依赖 hover。

3. **弹层层级统一**
   - 替换分散的 `z-index: 260 / 1000` 等魔法值。
   - 为下拉、右键菜单、搜索弹窗、Toast 建立统一 Sass 变量。

### P1：高收益优化

1. **Tailwind 与 Sass 令牌同步**
   - 现有 Tailwind 主题为空，后续会导致组件风格分叉。
   - 将 Arctic Blue 的颜色、字号、圆角、阴影同步到 Tailwind。

2. **搜索体验统一**
   - 局部搜索展开动画略长，焦点延迟偏保守。
   - 统一局部搜索与全局搜索的提示、快捷键、反馈方式。

3. **设置页信息分组**
   - 设置页功能密度较高，后续建议增加分组导航或锚点目录。
   - 首轮先统一卡片、按钮和下拉视觉，不做大规模结构拆分。

### P2：视觉精修

1. 减少 `transition: all`，改为明确属性过渡。
2. 强化空状态行动导向。
3. 统一按钮语义：`primary / secondary / ghost / danger / icon`。
4. 番茄钟与任务上下文建立更明显连接。

## 首轮实施范围

首轮改造聚焦低风险、高收益项：

1. 补齐设计令牌与层级变量。
2. 同步 Tailwind 主题，避免后续样式体系继续分叉。
3. 优化局部搜索的展开速度与焦点体验。
4. 提高任务卡片操作按钮可发现性，但不改变操作逻辑。
5. 替换主要上下文菜单和设置下拉的魔法层级值。
6. 修复已确认的可见乱码文案。

## 验收标准

1. 主界面仍保持 Arctic Blue、浅色、紧凑、无边框窗口风格。
2. hover 不产生上浮、抬升或布局位移。
3. 键盘 focus ring 清晰可见。
4. 局部搜索展开更快，焦点响应更自然。
5. 任务卡片常用操作更容易被发现。
6. 样式变量和 Tailwind 主题具备一致的设计基准。
7. 至少通过 `pnpm verify:agent:fast`。

## 后续重构建议

1. 将 `TodoList.vue` 拆分为任务 Shell、头部、列表区域、上下文菜单。
2. 将 `TodoItem.vue` 拆分为任务主体、元信息、操作区、描述区、子任务区。
3. 将设置页拆为分组组件，并为快捷键、数据、更新建立独立区域。
4. 建立基础 UI 组件：按钮、图标按钮、徽章、菜单、空状态、弹层容器。
5. 引入视觉回归截图或至少用浏览器手动检查 375px、768px、1024px、1440px 宽度。
