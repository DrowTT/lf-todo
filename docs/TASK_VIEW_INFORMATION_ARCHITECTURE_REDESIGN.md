# 任务视图信息架构改造方案

## 背景

当前问题不是系统能力太少，而是侧边栏顶部系统视图过多，挤压了用户自建分类。对于从印象笔记等工具迁移来的重度用户，侧边栏最重要的价值是呈现“我的结构”，而不是堆叠大量系统预设视图。

经过视觉评估，常驻筛选、排序栏虽然能承接未完成和高优先级等能力，但会额外占用主内容区高度，并让任务列表可视面积变小。因此本阶段选择更克制的方案：先把侧边栏恢复为极简三入口，不在主内容区增加常驻筛选栏。

## 使用者视角判断

### 真正需要固定在侧边栏的入口

- **全部**：全局兜底入口，用户迷路时可以回到这里。
- **暂存区**：快速捕获入口，新任务默认进入这里。
- **已归档**：历史入口，低频但需要明确可达。

### 暂不固定展示的入口

- **今天**：有价值，但会让系统视图区域继续膨胀；后续更适合放到命令面板、快捷入口或可配置视图。
- **未完成**：属于状态筛选，不适合长期占侧边栏。
- **高优先级**：属于优先级维度，不适合长期占侧边栏。
- **最近 / 未来**：属于排序或日期筛选维度，暂不进入默认导航。

## 本次实施原则

1. **先释放空间**：侧边栏顶部只保留三个最高必要入口。
2. **不压缩列表高度**：主内容区不新增常驻筛选、排序栏。
3. **保留现有搜索**：标题区搜索仍作为轻量查找能力存在。
4. **保留底层能力**：系统视图过滤能力暂不删除，避免影响已有状态和后续扩展。
5. **分类优先**：让自建分类更早进入视野，符合知识库迁移用户的组织心智。

## 本次实施范围

### 侧边栏

顶部“视图”区域只保留：

- 全部
- 暂存区
- 已归档

删除默认展示的：

- 今天
- 未来
- 未完成
- 高优先级
- 最近

### 主内容区

- 不新增常驻筛选栏。
- 不新增常驻排序栏。
- 保持标题、搜索、任务输入框和任务列表的纵向空间。
- 继续保持搜索时暂停拖拽排序，避免搜索结果视图误改真实顺序。

## 后续演进建议

1. 如果后续要恢复筛选能力，优先考虑“标题区一个漏斗按钮 + 弹出面板”，不要常驻占一整行。
2. “今天 / 高优先级 / 未完成”等可以做成可配置快捷视图，由用户决定是否固定到侧边栏。
3. 排序能力更适合放进任务列表右上角的单个图标菜单，而不是三个并列下拉框。
4. 对重度用户，可通过命令面板提供快速跳转和临时筛选。
5. 后续如需做保存视图，应允许用户自定义名称和是否固定，而不是默认塞进侧边栏。

## 验收标准

- 侧边栏顶部只出现全部、暂存区、已归档三个入口。
- 分类区域明显上移，减少被系统视图挤压的感觉。
- 主内容区没有新增常驻筛选、排序栏。
- 任务列表可用高度恢复。
- 搜索、拖拽、归档已完成等既有行为保持可用。
